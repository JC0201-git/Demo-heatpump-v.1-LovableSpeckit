## 釐清記錄

所有 [NEEDS CLARIFICATION] 項目已於 2026-05-23 確認，5 個話清問題已於同日全數解決：

1. **資料介接方式** → 已有 IoT 閘道器 / SCADA API；儀表板後端僅需串接現成 API
2. **使用者角色隔離** → v1 為內部員工共用後台；系統預留多租戶擴充介面，v1 不實作
3. **設備規模** → 預期總量 80 台；3 台真實資料，77 台虛擬資料；未指定場域者隨機配置
4. **風險指派方式** → v1 手動指派（高／中／低）；v2 升級為多指標加權自動評分
5. **角色存取權限** → 維運人員前 4 頁（含寫入）；高層管理者全 6 頁（唯讀）
6. **月報格式** → v1 僅 PDF 下載
7. **設備編號格式** → `場域代碼-流水號`（如 `SITE01-001`）；IoT 序號另存參考欄位
8. **設備狀態觸發條件** → 離線 = 5 分鐘無資料；警告 = 超警戒傀；異常 = 超危急傀或告警觸發
